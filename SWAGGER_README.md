## 🏗️ Microservices Overview

### 1. **Main Microservice** (Port 8000)
- **File:** `microservice.py`
- **Endpoints:** 6
  - ✅ `POST /submit_application` - Submit loan application
  - ✅ `GET /application_status/{application_id}` - Check application status
  - ✅ `GET /applications` - List recent applications
  - ✅ `GET /search_applications` - Search applications
  - ✅ `GET /application_details/{application_id}` - Get full details
  - ✅ `GET /health` - Health check

## 📋 Key Response Fields

### Application Response
```json
{
  "application_id": "CASE-APP-{applicant_id}-{timestamp}",
  "status": "approved|rejected|review|pending",
  "decision": { ... },
  "financial_risk": { ... },
  "applicant_profile": { ... },
  "compliance_action": { ... },
  "overall_reasoning": "string"
}
```

### Decision Levels
- **Approved:** Risk Score < 30
- **Review:** Risk Score 30-75
- **Rejected:** Risk Score ≥ 75

## 🛠️ Troubleshooting

### Port Already in Use
```bash
# Find process using port 8000
lsof -i :8000

# Kill process
kill -9 <PID>
```

### CORS Issues
- Add CORS headers to FastAPI app (already configured in main microservice)
- Check cross-origin requests

### Database Errors
- Ensure `loan_applications.db` exists
- Run `init_db()` if needed
- Check database permissions

## 📝 Notes

- All timestamps are in ISO 8601 format
- Application IDs follow pattern: `CASE-APP-{applicant_id}-{YYYYMMDDHHMMSS}`
- All numeric responses use float precision
- Error messages are descriptive and actionable
